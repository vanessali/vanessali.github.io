Time Shifting Phase Vocoder

A final project for Music and Computing course at McGill University.

Objectives
To build a phase vocoder in Max/MSP that allows for time compression and expansion while retaining the frequency and phase content.

Methodology
The phase vocoder was implemented in the frequency domain using the pfft~ object. An audio file is read into a buffer, which is then sent through the pfft~ object for analysis. The window size of my fft is 1024 samples with a factor of 16 overlap. The analyzed signal is recorded into a two-channel buffer which holds the amplitude and phase data of each frame. Each frame can then be accessed and resynthsized using a wavetable. This specific implementation was based on Cooper Baker's phase vocoder sampler patch.

The rate at which signal is played back is determined by a lookup table that indicates which frames are to be resynthsized. The available waveforms for the lookup table to choose from are a phasor (linear ramp from 0-1), a sinusoidal signal (a cosine shifted and scaled by a value of +0.5 to be confined within 0 and 1), a wavetable (user can choose from presets or draw their own non-linear waveform) and a selected region of audio, which the user can use to scrub through the buffered sample.
